# GitHub Pages移行チェックリスト

このサイトは、現在このリポジトリ内のファイルだけで表示できる状態です。

## フェーズ1: 公開前の固定化

1. ローカルで表示崩れがないか確認する
2. テキスト、画像、リンク先を最終確認する
3. GitHubへpushする前に不要ファイルがないか確認する

## フェーズ2: GitHub Pages仮公開

1. GitHubにリポジトリを作成する
2. このフォルダ一式をpushする
3. GitHubの Settings > Pages を開く
4. Source を Deploy from a branch にする
5. Branch は main、Folder は /(root) を選ぶ
6. 発行された github.io のURLで確認する

## フェーズ3: 仮公開URLで確認する項目

1. トップページがPCとスマホ幅で崩れないか
2. 画像がすべて表示されるか
3. ナビゲーションリンクが正しくスクロールするか
4. favicon が表示されるか
5. 日本語フォントの見え方に違和感がないか

## フェーズ4: ドメイン移行の準備

1. 現在のDNS管理サービスを確認する
2. 既存サイトのTTLを事前に短くできるなら短くする
3. GitHub Pages側で使う独自ドメイン名を確定する
4. CNAME を追加する直前まで github.io 側で最終確認する

## フェーズ5: ドメイン切替

1. GitHub Pagesの Custom domain に独自ドメインを設定する
2. DNSに必要なCNAMEまたはAレコードを設定する
3. HTTPS証明書の発行完了を待つ
4. 切替直後は旧サイトをすぐ消さず、名前解決の反映を監視する

## フェーズ6: 切替後の確認

1. 独自ドメインでトップページが開くか
2. www有り無しの揃え方が意図通りか
3. 画像、アイコン、CSS、JSがすべて200で返るか
4. 検索エンジン向けのURLが意図通りか

## 補足

- 今の段階では、独自ドメイン設定用の CNAME ファイルはまだ置いていません
- ドメイン名が確定してから追加する方が安全です
